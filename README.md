# Vendor Due Diligence Assistant

Multi-agent (LangGraph) system that reviews vendor-provided documents (security questionnaires, DPAs, policies) and produces a structured, evidence-backed security compliance assessment.

**Status: Milestones 1–5 complete — "Interactive Multi-Agent Review Pipeline"**

---

## 🛠️ Tech Stack
- **FastAPI (async)** — Backend API and execution orchestration layer.
- **LangGraph** — Multi-agent state orchestration workflow (Classifier, Retrieval, Risk, Gap, Contradiction, Summary nodes) with persistent DB state checkpointing.
- **Streamlit** — Graphical dashboard UI featuring real-time event logs, agent progress visualizers, custom instruction injectors, and in-flight grounded Q&A.
- **SQLAlchemy 2.0 (async) + Alembic** — ORM schema models and autogenerated database migrations.
- **PostgreSQL + pgvector** — Document metadata and semantic vector similarity search index.
- **JWT Auth** — Role-based access control (User/Admin).
- **uv** — Dependency and environment manager.

## 🚀 Local Setup

### Option A: With Docker (Recommended)
1. Start the PostgreSQL + pgvector containers:
   ```bash
   docker compose up -d
   ```

### Option B: Without Docker (Native PostgreSQL)
1. Install PostgreSQL 16 (from EDB) natively on your laptop. Set port as `5432` and set a password.
2. Install the **`pgvector`** extension:
   * Download the pre-built `pgvector` binaries for Windows/macOS.
   * Copy the `.dll` file from `lib/` to `C:\Program Files\PostgreSQL\16\lib\`.
   * Copy the `.sql` and `.control` files from `share/extension/` to `C:\Program Files\PostgreSQL\16\share\extension\`.
3. Create a blank database named `vendor_dd` via pgAdmin.

---

### Step 2: Install Python Dependencies
```bash
uv sync
```

### Step 3: Configure Environment Variables
Copy the template file to `.env`:
```bash
cp .env.example .env
```
Open `.env` and fill in your keys:
- Set `DATABASE_URL` (change port or password to match your setup).
- Input a real `OPENAI_API_KEY` (required for embeddings and agent prompts).

### Step 4: Run Migrations
Applies the full schema (including document chunks, findings, and agent runs) to your database:
```bash
uv run alembic upgrade head
```

### Step 5: Start the Backend Server
```bash
uv run uvicorn app.main:app --port 8000 --reload
```
- Swagger UI will be available at: http://localhost:8000/docs
- Seed Admin Account: `admin@vendordd.internal-app.com` / `change_this_admin_password_123!`

### Step 6: Start the Streamlit Dashboard UI (In a new terminal)
```bash
uv run streamlit run ui.py
```
- Open in browser: http://localhost:8501

## 🔬 Running the Integration Test Suite
To verify the complete backend API pipeline (upload, preprocessing, agent run, pause, resume, grounded Q&A) offline, run:
```bash
uv run python scripts/smoke_test_m4_m5.py
```
